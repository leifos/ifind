Experiment Instructions

First of all, thank you for agreeing to participate in this experiment about online searching!
During this experiment we would like you to perform three different search tasks using the same search system.

Your Role

  • For this experiment, imagine that you are a newspaper reporter, where you will need to write a story on three topics. In order to accomplish this you will need to search for related information.

The Search System

  • The search system that you will use connects to a database of about 1 million newspaper articles from the New York Times, The Associated Press and Xinhua news agencies. The articles within the database range from 1996 to 2000. The search system you'll be using looks similar to a web search engine, though there might be some slight differences to the search interface.

The Search Tasks

  • For each search task, you'll be given a short description of a topic and what information would be relevant to this topic.

  • You will begin posing queries and looking at results, just like you would in a web search engine. When looking at documents, you can mark whether a document is relevant to the given topic by clicking on a button. There are over 100 relevant documents in the database for each topic. Your job is to find as many of these as possible during the time allocated (approximately 20 minutes per task).

Experiment Outline

Before you start searching, we will ask you a few questions about you.

We will then give you a few minutes to get comfortable with the search interface by undertaking a practice task.

You'll then move onto the two main search tasks. Before each task, we will ask you a few questions about your knowledge of the search topic. After you complete the task, we will ask you some questions about your search performance.

After you have performed each of the search tasks, we will ask you to complete a series of very short questionnaires about your search experience using the system provided to perform all the search tasks.
